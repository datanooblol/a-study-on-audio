Study plan

AI + Audio

A self-study curriculum on how AI is applied to audio: understanding it (speech-to-text, diarization, speaker ID, emotion) and generating it (text-to-speech, voice cloning) — built on a shared foundation of digital signal processing.

How this curriculum works

Phase 1 (this repo, right now): eleven conceptual lessons, each a self-contained HTML page with diagrams, history, and the evaluation metric practitioners actually use. No code yet — the goal is a correct mental model of each task before touching a library.

Phase 2 (later): once the lesson set feels solid, we deep-dive into code — notebooks that implement each pipeline.

Written for a data scientist who already knows ML/AI fundamentals but is new to the audio-specific parts: sampling, spectrograms, vocoders, and the metrics unique to speech and audio tasks.

Learning path

Four stages, in teaching order. Later modules lean on representations introduced in earlier ones.

Why this order: foundations (sampling theory, safe format conversion, preprocessing, feature extraction) feed every task that follows, so they come first — format conversion specifically sits right after the sampling-theory lesson, since you need to know how to get audio into a target format safely before preprocessing or any model touches it. Within "understand audio," tasks are ordered by how directly they reuse the previous one's output (ASR → diarization → verification all lean on similar embeddings; emotion recognition closes out analysis since it can consume a transcript too). Generation comes last because TTS and voice cloning are the most complex pipelines, and voice cloning specifically builds on TTS.

01 · Foundations of Digital Audio

How sound becomes numbers — sampling, quantization, and the time/frequency duality that every downstream model relies on.

  • Sample rate, bit depth, Nyquist frequency
  • Waveform vs. spectrogram — reading both, knowing when to use which
  • The Fourier Transform / STFT, intuitively (no heavy math)
  • Audio file formats, and lossy vs. lossless codecs
  • Choosing a sample rate, bit depth, and format for a task — and what real-world datasets actually ship as
TaskTypical sample rateBit depthFormat
ASR / STT16 kHz16-bitWAV (PCM) or FLAC
TTS / vocoder training22.05–24 kHz (up to 48 kHz for HiFi vocoders)16-bitWAV
Diarization / speaker ID16 kHz16-bitWAV
Music / general audio44.1–48 kHz16–24-bitWAV or FLAC
Archival / source recordingshighest available (48–96 kHz)24-bitWAV or FLAC, lossless — downsample per task later, never upsample from a lossy source

Rule of thumb: match the sample rate a pretrained model was trained on (most speech models standardize on 16 kHz) rather than assuming "higher is better" — resolution a model was never trained on is wasted storage, not free accuracy.

02 · Audio Formats & Conversion

The practical map of containers and codecs you'll actually run into, and how to move audio between them without silently degrading it.

  • Container vs. codec — why .wav and .mp3 aren't the same kind of thing, and where FLAC / OGG / M4A / AAC / Opus fit
  • Lossless vs. lossy, and why repeated lossy→lossy re-encoding compounds quality loss ("generation loss")
  • Safe sample-rate conversion (proper anti-aliasing resampling) vs. naive resampling artifacts
  • Bit-depth conversion and dithering when going from 24-bit to 16-bit
  • Stereo→mono downmixing pitfalls (phase cancellation)
  • A do/don't checklist for round-tripping audio through multiple tools and pipelines
DoDon't
Keep one lossless master; convert working copies from itRe-encode an already-lossy file into another lossy format "to save space"
Resample with a proper filter (ffmpeg / sox / librosa's resampler)Naively drop or duplicate samples to change rate
Dither when reducing bit depthTruncate bits and call it done
Check sample rate / channel count match what a pretrained model expectsAssume "it's audio, any format works"

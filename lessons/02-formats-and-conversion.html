02 · Audio Formats & Conversion

The practical map of containers and codecs you'll run into, and how to move between them without silently degrading the signal.

Overview

In practice, audio never arrives in the format your model wants. A dataset ships as FLAC, your model wants 16 kHz mono WAV; a scraped clip is an MP3; a phone recording is 8 kHz µ-law. Converting between formats looks trivial — one CLI flag — but done carelessly it quietly destroys quality in ways that don't show up until your model's accuracy drops and you can't tell why. This lesson is the map and the checklist.

Key concepts

Container vs. codec

A container is the file wrapper (metadata + one or more encoded streams). A codec is the algorithm that actually encoded the audio inside it. The same extension can hide different codecs, which is the single biggest source of confusion.

Lossless vs. lossy

Lossless — WAV (PCM), FLAC, ALAC

Stores (or perfectly reconstructs) the exact samples. Larger files, but decoding and re-encoding never loses information. Safe to convert back and forth freely.

Lossy — MP3, AAC, Opus, Vorbis

Throws away detail a psychoacoustic model predicts you won't notice, to shrink file size. That decision can't be undone — decoding a lossy file never recovers what was discarded.

Generation loss

Re-encoding lossy → lossy (MP3 → AAC → MP3, or even MP3 → MP3 at a different bitrate) compounds artifacts each pass — like photocopying a photocopy. If a source file is already lossy, decode it once and derive everything else from that decode; never chain lossy re-encodes.

Sample-rate conversion (resampling)

Changing the sample rate (e.g. 44.1 kHz → 16 kHz) means computing new sample values at new time positions. Done right, this applies a low-pass anti-aliasing filter before discarding samples (downsampling) or an interpolation filter when adding them (upsampling). Done naively — just dropping or duplicating samples — it lets frequencies above the new Nyquist limit "fold back" as false low frequencies (aliasing), corrupting the signal in a way that's hard to undo.

Upsampling doesn't add information

Converting 8 kHz audio to 16 kHz produces a file that plays back correctly, but it doesn't recover detail that was never captured. Use it for compatibility, not for quality — the real fix is finding a source that was captured at the higher rate.

Bit-depth conversion & dithering

Going from 24-bit to 16-bit means representing each sample with less precision. Simple truncation introduces quantization error that's correlated with the signal, which the ear perceives as harsh distortion. Dithering adds a tiny, controlled amount of noise before truncating, decorrelating that error from the signal — trading a nearly inaudible noise floor for the removal of harsher, structured distortion.

Channel conversion (stereo → mono)

Most speech/ML pipelines want mono. Naively averaging left and right channels is usually fine, but if the two channels are out of phase (common in some processed music or multi-mic setups), averaging can cancel shared frequencies and leave a thin, hollow-sounding result. Worth a quick listen after downmixing anything that isn't already single-channel speech.

A little history

  • 1991 — WAV / RIFFMicrosoft and IBM's uncompressed PCM container becomes the baseline every other format gets compared against.
  • 1993 — MP3The first lossy codec good enough for mainstream listening, using a psychoacoustic model to discard sound you're unlikely to notice missing — it made internet audio practical.
  • 1997 — AACDesigned as MP3's successor: better quality at the same bitrate, later the default for Apple's ecosystem and streaming.
  • 2001 — FLACAn open, free, lossless codec — compresses like a zip file (losslessly) rather than discarding anything, becoming the standard for archival and dataset distribution.
  • 2012 — OpusAn IETF standard tuned for both real-time, low-latency communication (VoIP, WebRTC) and high compression efficiency — now common in streaming and voice pipelines.

Evaluation

No standard metric — sanity-check conversions instead: compare a spectrogram of the source and the converted file side by side (aliasing shows up as unexpected energy after conversion), check peak levels for new clipping, and do a quick listen for artifacts.

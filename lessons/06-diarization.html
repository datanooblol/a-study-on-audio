06 · Speaker Diarization

"Who spoke when" — independent of what was actually said.

Overview

Diarization answers a question ASR doesn't: in a recording with multiple people, which stretches of audio belong to which speaker? It doesn't need to know who the speakers are (that's verification/identification, lesson 07) — just where "Speaker A" stops and "Speaker B" starts. It's what turns a raw meeting recording into a readable, per-speaker transcript.

Key concepts

The standard pipeline: VAD → embed → cluster

  1. VAD finds the regions that contain speech at all (lesson 03).
  2. Those regions are cut into short windows, and a speaker embedding is extracted for each one (the same kind of embedding used for speaker verification — a fixed-length vector that captures voice characteristics, not words).
  3. Embeddings are clustered (commonly agglomerative hierarchical clustering) so that windows from the same speaker end up in the same group — without necessarily knowing in advance how many speakers there are.
  4. Cluster assignments are mapped back onto the timeline: "Speaker 1: 0:00–0:12, Speaker 2: 0:12–0:19, …"

Overlapping speech: the hard case

Classic diarization assumes one active speaker per moment. Real conversations interrupt and talk over each other constantly. A pure clustering pipeline can only assign one label per time window, so overlaps get mis-attributed or smeared across a boundary. Two ways around it:

Overlap detection

A separate detector flags overlapping regions so the system can at least mark "two speakers here" rather than silently guessing one.

End-to-end neural diarization (EEND)

A single network directly predicts, per time step, which of several speakers are active — naturally allowing more than one "yes" at once, without a separate clustering step.

Pairing diarization with a transcript

Diarization and ASR usually run as two separate systems that need to be merged: align the diarization timeline's speaker-labeled intervals with the ASR transcript's word-level timestamps to produce a readable "Speaker A: ... / Speaker B: ..." transcript. Small timing mismatches between the two systems are a common source of misattributed words at speaker-change boundaries.

A little history

  • 2000s — BIC segmentation + GMM clusteringEarly diarization (largely for broadcast news, evaluated in NIST Rich Transcription benchmarks) used the Bayesian Information Criterion to find speaker-change points, then clustered segments with Gaussian Mixture Models.
  • 2010s — i-vectorsA compact speaker-representation technique from speaker verification research gets adapted to diarization, improving robustness.
  • ~2017 — x-vectorsDeep neural embeddings ("x-vectors") replace i-vectors as the standard speaker representation, driven by the same embedding advances used in verification (lesson 07).
  • 2019+ — End-to-end neural diarizationEEND-style models learn to output multi-speaker activity directly, finally handling overlapping speech natively rather than as an afterthought.
  • 2020 — pyannote.audioAn open-source toolkit packages these ideas into pretrained, easy-to-use pipelines, becoming a de facto standard for applied diarization work.

Evaluation

DER (Diarization Error Rate) combines three error types over total reference speech time:

DER = (False Alarm + Missed Speech + Speaker Confusion) / Total reference speech time

  • False alarm — non-speech labeled as speech
  • Missed speech — speech labeled as non-speech
  • Speaker confusion — speech correctly detected but attributed to the wrong speaker
